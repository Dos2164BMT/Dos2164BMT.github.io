C. Category Three: Databases

i. Artifact Selection, Category, and Origin

For the databases category, I selected my CS-465 Travlr Getaways full stack project. This artifact was created during my CS-465 Full Stack Development course. The project uses the MEAN stack, including MongoDB, Express, Angular, and Node.js, to manage travel package data for a web application.

I selected this artifact because it connects database development with full stack application design. MongoDB is used to store travel package information, while Express and Node.js support the back-end application logic. Angular supports the front-end interface that displays and interacts with the data.

ii. Practical Enhancement Plan With Pseudocode

My enhancement plan is to improve how travel package data is stored, retrieved, updated, and displayed. I plan to improve the MongoDB schema design, add stronger validation for travel package records, improve API routes for database operations, and make sure the application handles database errors clearly.

This enhancement will also include improving CRUD functionality so travel package data can be created, read, updated, and deleted more reliably. I also plan to avoid hardcoded sensitive information and improve the structure of the database connection logic.

Pseudocode

Start Application
Connect to MongoDB database

If database connection is successful:
    Load travel package schema
    Start Express server

When user requests travel packages:
    Send GET request to API
    Query MongoDB for travel packages
    Return package data to front end

When user creates or updates package:
    Validate package fields
    If data is valid:
        Save or update package in MongoDB
        Return success message
    Else:
        Return validation error message

When user deletes package:
    Confirm package exists
    Delete package from MongoDB
    Return result message

If database error occurs:
    Log error
    Return clear error response

End Application
